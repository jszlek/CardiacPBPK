1. Introduction

Prediction of drug effect is valuable in terms of both safety and efficacy. Modern model-based approaches can be useful in drug concentration-effect establishment, preceded by the estimation of active concentrations of drugs. With specifically cardiac safety and efficacy assessment in mind, we developed a mechanistic physiologically based pharmacokinetic (PBPK) model of the heart useful in assessment of the cardiac active drug fraction surrogate responsible for the clinically observed drug-related electrophysiological response in humans. The PBPK models were developed for a parent compound and its metabolite and include inter-subject variability. The models were written in R v.3.4.0. The detailed description of the models and the R code were published in Tylutki Z., Mendyk A., Polak S. J Pharm Sci, 2018, 107(4):1167-1177. Model performance verification was done for amitriptyline and its main metabolite – nortriptyline by a physiologically based pharmacokinetic-quantitative systems toxicology and safety (PBPK-QSTS) modeling approach. The results of this PBPK-QSTS modeling were published in Tylutki Z, Mendyk A, Polak S. J Pharmacokinet Pharmacodyn, 2018, 45(5):663-677. Now we have decided to share the model as an web application to allow its further testing and usage as we hope that it would be useful for other users. However the application is distributed without any warranty of merchantability or fitness for a particular purpose.

The model is the result of a PhD project defended by Zofia Tylutki at the Jagiellonian University, Krakow, Poland written under the supervision of Sebastian Polak, PhD.

This is a User Interface (ui) application which implements PBPK-QSTS models. Please visit the page, https://github.com/jszlek/PBPK-QSTS for more information on the application.


2. Installation

Requirements:

1) R environment version >= 3.3.0 (please follow the instructions devoted to your OS in order to install R),

2) Additional R packages: shiny, shinyjs, DT, shinyBS, dplyr, plyr, ggplot2, ggthemes, distr, data.table, deSolve (please follow the instructions at r-bloggers.com)


Installing PBPK-QSTS:

1) Download the tar.gz archive from the https://sourceforge.net/projects/ or clone github repository (if you have git packages installed in konsole/terminal type ‘git clone https://github.com/jszlek/PBPK-QSTS.git’). It is also possible to download master branch from github (https://github.com/jszlek/PBPK-QSTS/archive/master.zip),

2) Unpack/decompress downloaded package (on linux-like systems type ‘tar xzvf (...).tar.gz -C /path/to/my/.../folder’),

3) Run the konsole/terminal or cmd.exe,

4) Navigate to the folder (on linux-like systems use ‘cd /path/to/my/.../folder’),

5) On Windows run the run_R.bat file or on linux-like systems run run_R.sh (make sure the *.bat or *.sh files are executable and R installation folder is in the $PATH environment variable)


Alternatively:

5) Run R console (on linux type ‘R’, on Windows look for R in Start menu)

6) Check if you are in the folder by typing ‘getwd()’ in R console

7) Load shiny library by typing ‘library(shiny)’ - if you do not have it installed please do so.

8) Run the shiny application by typing ‘runApp()’

9) This command should run the app in a browser


3. Running simulations

The main window of the program is divided into six tabs: Welcome, Population, Study, API, Run, About. Welcome and About tabs contain general information on the program usage as well as license.




Before running simulations you must carefully chose the parameters gathered in tabs Population, Study and API. Tab Run is used to start simulations, view and save the results.

Population tab is divided into three sub-tabs: Settings, Dataset, Plots. The last two are for viewing and saving the dataset. Moreover the histograms of basic population parameters are shown. The Population settings tab is ordered into three columns: General parameters, Absorption parameters and Distribution parameters. General parameters are responsible for setting: seed, which will be used through whole simulation, number of individuals, female percentage, age range, scale and shape parameters for age distribution. It is also possible to set liver and heart density together with pericardium blood flow.




Study tab is used to modify the parameters such as: the amount and number of doses. Moreover here you can select the duration of simulation.




API tab is designed to modify drug parameters such as: physico-chemical, distribution and elimination related. This tab is suitable to set partition coefficients for various tissues, elimination constant (Km) and volume of distribution (Vmax). Here you can also chose if the simulations should include calculations for metabolite of the drug.




The last tab – Run – contains the button ‘Run’ which starts the simulation process. After a while, considering the number of subjects used, the plots of concentrations vs. time in different compartments and results table are shown.
